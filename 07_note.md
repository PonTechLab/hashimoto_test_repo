## ブランチ
履歴の枝分かれ。  
「変更後のファイル」というよりも変更箇所をコミットごとに記録したもの。  
ファイル丸ごと保存しているわけではないので容量はとても軽い。  
- git branch   
今のブランチを確認
- git branch 名前  
新しいブランチを作成
- git　branch -d  
ブランチの削除 

## git switch
作業ファイルの状態をどのブランチのものにするか選んで切り替える。  
- git switch -c 名前  
新規ブランチ作成＆そこに移動。  
似たコマンドにcheckoutもある。  
checkoutは作業ディレクトリの変更やファイル状態の復元もできる。  
多機能ゆえに「作業中のファイルを過去の状態に戻したくないのに戻してしまった」ということも起きるので機能を分割した。  
現在はswitchが主流。  
- git restore  
間違ってaddしたファイルを元に戻したい時に使う（ステージングエリアにあるファイルをワーキングエリアに戻すとき）

## git merge
別のブランチの変更を現在のブランチに反映すること。
自分のPC内で作業中のブランチと別なブランチを合わせるときに使う。

※git fetch  
リモートリポジトリの最新情報を取得して、追跡リポジトリにコピーすること


## git pull
Fetchとmergeの機能を合わせたもの。
リモートリポジトリから最新の情報を持ってきて自分のブランチに合流させる


## プルリクエスト
自分の書いたコードをmainブランチに合流させていいかお伺いすること。
レビューをもらう場。
